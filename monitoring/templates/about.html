Project Overview

This project bridges the physical world with the digital by collecting environmental data using sensors and displaying it online through a dynamic Django-based website.

Components Used

ESP32 Microcontroller

Our central processing unit for handling sensor inputs and communication.

DHT11 & MQ135 Sensors

Used for collecting temperature, humidity, and air quality readings.

Veroboard

We designed and soldered our custom circuit using a veroboard for better reliability and compactness.

Wi-Fi Module

ESP32 Built-in - For transmitting sensor data to the Firebase database and then display it on the server.

Django Framework

Built the web application that receives, stores, and displays data.

Firebase Realtime Database

Used for real-time data storage and synchronization across the network.

Frontend Technologies

HTML/CSS/JavaScript - For a responsive and visually intuitive frontend display.

Project Workflow - Step by Step

1

Planning & Design

We started by outlining the project's scope: reading environmental data using sensors and displaying it live on a website. We chose the ESP32 due to its built-in Wi-Fi module and Django for its powerful backend capabilities.

2

Hardware Setup

ESP32 and sensor connections were soldered on a veroboard, ensuring compact layout and proper pin alignment. The DHT sensor and MQ135 sensor was connected with GPIO 4 and 34 respectively, and the system was powered via USB cable (connected via laptop or external input).

3

Sensor Integration

Embedded C/Python code was written for the ESP32 to fetch and throw humidity, temperature and air quality data at regular intervals and send it to our server using Firebase Realtime Database, HTTP requests and much more.

4

Frontend Development

We worked on building the frontend using HTML, CSS, and online libraries and extensions, along with JavaScript to create live-updating sections for temperature and humidity. A professional and impressive frontend with amazing ui and responsiveness was ensured, which was user-friendly.

5

Backend Development

Development of the Django backend — creating models to store sensor data, setting up APIs endpoints to receive incoming data, and implementing data validation and time-stamping.

6

Testing & Debugging

The system was tested extensively — checking for data delays, sensor accuracy, UI glitches, and backend and frontend issues along with database connectivity issues. Also iterative improvements were made until the data flow and site performance were smooth.

7

Deployment & Hosting

The website was hosted online (either locally or via a server like Ngrok which works as a reverse proxy server), and the ESP32 was configured to send/recieve data from the hosted domain/server IP.

8

Final Integration

We finalized the styling, ensured data synced in real-time, and confirmed that all components were stable and reliable.
